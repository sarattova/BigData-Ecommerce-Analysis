1. Актуальность и цель
В условиях современной цифровой экономики объемы данных о покупках растут экспоненциально.
Цель данного проекта — применить изученный инструментарий больших данных для извлечения полезной информации из сырых логов продаж.
Это позволяет автоматизировать отчетность и находить скрытые закономерности, которые невозможно обработать вручную в Excel.

2. Используемый инструментарий
HDFS: Хранение исходного датасета в распределенной файловой системе.
Apache Spark (PySpark): Основной движок для обработки данных (DataFrame API).
Matplotlib / Seaborn: Для финальной визуализации результатов.

3. Этапы реализации
Прием данных: Загрузка CSV-файла с Kaggle в HDFS. Определение схемы данных (типы данных для цен, дат и количеств).
Обработка и очистка:  Удаление строк с пустыми значениями (например, если нет ID пользователя).
Преобразование форматов дат из строк в объекты Timestamp
Фильтрация аномалий (например, отрицательные значения количества товара).
Аналитический блок:
Расчет ABC-анализа: определение 20% товаров, которые приносят 80% выручки.
Временной анализ: поиск часов самой высокой нагрузки на систему (когда люди чаще всего покупают).
Географический анализ: распределение заказов по странам или городам.
Визуализация: Построение графиков, отображающих динамику продаж и топ-категории товаров.
